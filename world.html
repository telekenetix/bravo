world – Worlds¶

class beta.world.World(folder)¶

Object representing a world on disk.

Worlds are composed of levels and chunks, each of which corresponds to exactly one file on disk. Worlds also contain saved player data.

generate_level()¶

Generate the level metadata.

This method currently does not generate all of Alpha’s metadata, just the data used by Beta.

load_chunk(x, z)¶

Retrieve a Chunk.

This method does lots of automatic caching of chunks to ensure that disk I/O is kept to a minimum.

load_level_data()¶

Load level data.

load_player(username)¶

Retrieve player data.

populate_chunk(chunk)¶

Recreate data for a chunk.

This method does arbitrary terrain generation depending on the current plugins, and then regenerates the chunk metadata so that the chunk can be sent to clients.

A lot of maths may be done by this method, so do not call it unless absolutely necessary, e.g. when the chunk is created for the first time.

save_off()¶

Disable saving to disk.

This is useful for accessing the world on disk without Beta interfering, for backing up the world.

save_on()¶

Enable saving to disk.

saving¶

Whether objects belonging to this world may be written out to disk.

season¶

The current ISeason.
